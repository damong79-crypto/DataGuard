# DataGuard 사용자 매뉴얼

DB 테이블 정합성을 등록한 쿼리로 자동 점검하고, 이상이 발견되면 이메일로 알려주는 Windows 데스크탑 앱입니다.

> 대상 독자
> - **등록자**(DBA·데이터 엔지니어): 앱에서 연결·쿼리·스케줄을 관리하는 사람
> - **수신자**: 결과 이메일만 받는 사람 (앱을 쓰지 않음)

---

## 1. 설치

1. 배포받은 **`DataGuard.App.exe`** 를 원하는 폴더에 둡니다. (별도 설치 과정 없음)
2. 더블클릭하여 실행합니다.

> 처음 실행 시 Windows의 **SmartScreen / Smart App Control**이 "보호됨" 경고를 띄울 수 있습니다.
> 서명되지 않은 사내 도구라서 그렇습니다. `추가 정보 → 실행`을 누르거나, 사내 정책에 따라 허용하세요.
> (자세한 내용은 6장 참고)

설정·이력 데이터는 다음 위치에 저장됩니다:
`%LOCALAPPDATA%\DataGuard\` (config.json, history.db, 자격증명 폴더)

---

## 2. 기본 흐름 한눈에 보기

```
① 연결 추가  →  ② 체크 쿼리 추가  →  ③ 지금 실행(검증)  →  ④ 스케줄·이메일 설정
```

화면은 4개 탭으로 구성됩니다: **연결 / 체크 쿼리 / 이력 / 설정**.

---

## 3. 설정하기

### 3.1 DB 연결 추가

1. 상단 **연결 추가** 버튼 클릭
2. 입력:
   - **이름**: 식별용 (예: `운영-주문DB`)
   - **DB 종류**: SQL Server 또는 PostgreSQL
   - **호스트 / 포트 / 데이터베이스 / 사용자 / 비밀번호**
     - 포트를 비우면 기본 포트 사용 (SQL Server 1433 / PostgreSQL 5432)
3. **연결 테스트** 버튼으로 접속 확인 (✓ 성공 / ✗ 실패 + 원인)
4. **저장**

> 🔒 비밀번호는 Windows 자격 증명 저장소(DPAPI)에 **암호화 저장**되며, 설정 파일에는 들어가지 않습니다.
> 점검 대상 DB는 **읽기 전용 계정** 사용을 권장합니다.

연결을 고친 뒤 **편집**(비밀번호 칸을 비우면 기존 값 유지) / **삭제**할 수 있습니다.
단, 그 연결을 사용하는 쿼리가 있으면 삭제가 막힙니다 — 해당 쿼리를 먼저 정리하세요.

### 3.2 체크 쿼리 추가

1. **체크 쿼리** 탭 → 상단 **쿼리 추가**
2. 입력:
   - **이름** / **대상 연결** / **발송 정책**
   - **스케줄**: 수동 실행만 / 매 N분 / 매일 HH:mm / 평일 HH:mm
   - **SQL**: **"틀어진 행만 반환"하는 쿼리**로 작성
3. **저장**

> ✅ 판정 규칙: **결과 0건 = 정상**, **1건 이상 = 이상**
> 그래서 SQL은 "문제가 있는 행만 골라내는" 형태여야 합니다.
>
> 예시 (주문-결제 금액 불일치):
> ```sql
> SELECT o.order_id
> FROM orders o JOIN payments p ON o.order_id = p.order_id
> WHERE o.amount <> p.amount
> ```

### 3.3 이메일(SMTP)·수신자 설정

1. **설정** 탭 → **SMTP / 수신자 설정...**
2. SMTP 호스트·포트·SSL·사용자·발신 주소 입력, **수신자**는 한 줄에 한 명씩
3. 비밀번호는 빈칸으로 두면 기존 값 유지
4. **저장**

> SMTP 호스트가 비어 있으면 이메일 발송이 **비활성**됩니다 (실행·이력은 정상 동작).
> 메일 본문에는 **건수·요약만** 담기며 개인정보·실제 데이터 값은 포함되지 않습니다.

**발송 정책** (쿼리별 선택):
- **이상일 때만**: 문제(1건 이상) 발견 시에만 발송
- **항상**: 정상이어도 "이상 없음" 리포트 발송
- (실행 오류는 정책과 무관하게 항상 알림)

---

## 4. 실행과 모니터링

### 4.1 지금 실행 (수동)

**체크 쿼리** 탭에서 쿼리를 선택하고 상단 **지금 실행** → 즉시 점검, 결과가 이력에 기록되고 정책에 따라 메일 발송.

### 4.2 자동 실행 (스케줄)

스케줄이 설정된 쿼리는 **앱이 켜져 있는 동안** 정해진 시각/주기에 자동 실행됩니다.

> ⚠️ 상주형 동작입니다 — **앱이 꺼져 있거나 PC가 절전이면 그 시간대 점검은 건너뜁니다.**
> 무인 시간대(새벽 등)에도 돌려야 한다면 PC·앱을 켜둔 상태로 유지하세요.

### 4.3 트레이 상주

- 창의 **X 버튼**이나 **최소화**를 누르면 종료되지 않고 **트레이로 숨습니다** (백그라운드 계속 실행).
- 트레이 아이콘 **더블클릭** 또는 우클릭 **열기**로 다시 엽니다.
- 완전히 끄려면 트레이 우클릭 → **종료**.
- 이상·오류가 발견되면 트레이 **풍선 알림**이 뜹니다.

### 4.4 시작 시 자동 실행

**설정** 탭 → **"Windows 시작 시 자동 실행 (트레이로 시작)"** 체크 → 로그인 시 트레이에서 자동 시작.

---

## 5. 이력 관리

**이력** 탭에서 모든 실행 결과(쿼리·실행시각·상태·건수·소요시간)를 확인합니다.

- **필터**: 쿼리 / 상태(정상·이상·오류) / 기간(24시간·7일·30일) 선택 후 **조회**, **초기화**로 리셋
- **CSV 내보내기**: 현재 조회된 목록을 CSV 파일로 저장 (엑셀에서 한글 정상 표시)
- **자동 정리**: **설정** 탭에서 보관 기간(일) 지정 → 앱 시작·저장 시 기간이 지난 이력 자동 삭제 (`0` = 무제한)

---

## 6. 문제 해결

| 증상 | 원인 / 해결 |
|------|------------|
| 실행 시 "보호됨"·차단 경고 | 서명되지 않은 exe. `추가 정보 → 실행`. Smart App Control이 막으면 사내 정책에 따라 허용 또는 코드 서명 필요 |
| 연결 테스트 실패 | 호스트·포트·계정·방화벽 확인. 메시지에 원인 표시(마우스를 올리면 전체 표시) |
| 이메일이 안 옴 | 설정 탭에서 SMTP 호스트·수신자가 채워졌는지, 발송 정책이 "이상일 때만"인지 확인 |
| 스케줄이 안 도는 듯 | 앱이 트레이에 떠 있는지 확인(닫으면 안 됨). PC 절전/종료 시간대는 건너뜀 |
| 비밀번호를 잊음 | 보안상 조회 불가 — 연결 **편집**에서 새로 입력 |

문의: 사내 개발팀 / 저장소 이슈 등록.
